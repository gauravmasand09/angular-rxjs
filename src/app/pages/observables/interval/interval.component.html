Interval

interval returns an Observable that emits an infinite sequence of ascending integers, with a constant interval of time of your choosing between those emissions. The first emission is not sent immediately, but only after the first period has passed. Learn more


Timer

Creates an observable that will wait for a specified time period, or exact date, before emitting the number 0. timer(delay,interval) -delay means after how many seconds you want this to start. Learn more


Since interval waits for the passed delay before starting, sometimes that's not ideal. You may want to start an interval immediately. timer works well for this. Here we have both side-by-side so you can see them in comparison.

timer(0, 1000).subscribe(n => console.log('timer', n)); -- here timer will start immediately
interval(1000).subscribe(n => console.log('interval', n)); -- interval will start after delay of 1000 ms.
